廖雪峰Js基础学习——简记

发表于 2016-01-04   |   分类于 JavaScript

前言：

  • 一门脚本语言，电脑，平板，手机上浏览的所有网页，以及无数的基于HTML5的app基于JS驱动。
  • 跨平台，跨浏览器驱动网页

简介：

  • 创始人：Brendan Dich，两周的设计时间（1995年，网景公司）
  • ECMA（European Computer Manufacturers Association）为JavaScript定制了ECMAScript标准，前者是后者的实现。可以简单的理解为同一种东西。
  • 2015年6月，发布了ES6，可以简单的理解为js6版本，将ES6的版本看作是Java的版本，就会觉得很容易理解了。

快速入门：

  • Js代码可以在嵌入到网页的任何地方，一般放到标签中，如：

  • 也可以通过使用src属性来引入需要被执行的js文件 (维护方便，引用方便，可以参考android的标签)
    可以使用type属性来指明script标签的类型，默认是“text/javascript”

  • 使用chrome调试Js代码：

    • command+alt+i,打开或关闭开发者模式（上下分屏看源码）
    • command+alt+j,打开或关闭开发者模式下的js控制台

基本语法：

浏览器中负责执行的Js引擎会在每个语句末尾补上；，因此Js不强制要求句尾加；

  • 允许层级嵌套，过深需要抽取函数。
  • 不建议一行写多个语句。
  • 单行／多行注释等同于Java
  • 严格区分大小写

数据类型、变量

  • Number, Js不区分整数浮点数，如：123 0.456 1.23e4 －99
    NaN（无法计算结果时表示） Infinity 表示无限大，超过了Number的最大值。（可以做加减乘除取余。注意：2/0 ＝ Infinity， 0/0 ＝ NaN）
  • 字符串是实用单引号或者双引号括起来的任意文本
  • 布尔值 要么true， 要么false，也可以使用代数表达式来计算表示
  • &&,||,!,比较运算
  • ＝＝ 转换数据类型之后，再比较(存在诡异的bug)； ＝＝＝不自动转换数据类型，数据类型不一致，返回false， 否则进一步比较数值。推荐使用＝＝＝
  • NaN和谁都不等，包括自己，只有通过isNaN(NaN)来判断
  • 浮点数比较，1 / 3 === (1 - 2 / 3); // false 计算机计算浮点数时会有误差，因此只能通过计算两个数的差值是否小于某阀值来判断是否相等。
  • null表示一个空值，undefined表示值未定义。多数情况下使用null
  • 数组 Js的数组可以包含任意类型的数据 [1,2,3,7,’Hello’,null, true,false];
    • 使用 new Array(1,3,4,’ddd’); ========= [1,2,3, ‘fdsf’];
  • var arr = [1,3,true, ‘Bob’]; var用来声明变量，其中arr[4]是 undefined，这下知道undefined怎么用了吧
  • 对象，Js对象由一组无序的键值对来表示，如： var person ＝ ｛name: ‘Pober’, gender: ‘male’, age: 20 ｝;其中键均为字符串类型，值为任意类型。其中字符串如果是标准的标识符，则可以不加双引号，譬如存在－连字符的就不是标识符。
    • 通过.来获取对象中的内容，如 person.name
  • 变量 使用var声明，注意关键字的避规
    • Js中使用＝进行赋值，可以把任意数据类型赋值给变量，可以反复赋值重复利用。变量本身类型不固定，称之为动态语言。Java、Cplus，C为静态语言。

Strict模式
在Js中，如果不使用var声明变量，就直接使用一个变量，那么默认它为全局变量。使用var声明的变量的作用域则是对应的函数体内。可以使用 use strict放到js代码的第一行，如果浏览器支持strict模式，那将认为该没有var声明的变量存在ReferenceError错误。否则浏览器将会认为它是一个普通的字符串.
如果在同一区域重复对同一个变量做var声明，那么后者如果有值，则覆盖前者，否则不会影响前者变量的值

字符串

  • 使用‘’或“”括起来的字符串
  • 字符串中包含‘，可以使用“”括起来，如果包含“，就使用\转义，如：“I’m a \”Male\””,这样程序就会认为转义后的”为一个普通字符
  • \x## 表示Ascii码值为十六进制##的字符。如’\x41’＝‘A’;
  • ES6中，使用反引号...来代替‘\n’换行。如下：
    • `it
    • is
    • an
    • end`
  • 数组中，发生越界，只会返回undefined，并不会报错
    常见的字符串操作
  • s.length 求字符串长度,给其他类型数据求长度返回undifined
  • 和Java一样，字符串具有不可变性，给使用索引获取到的某一个字符赋值没有意义。
  • s.toUpperCase();／toLowerCase返回一个新的字符串
  • indexOf(‘word’); 返回字符串’word‘在字符串中出现的位置，结果是w所在的索引
  • substring(0,5); 同样的，左闭右开。
  • substring(7); 返回从7开始到结束的字符
  • 可以直接使用‘＋’来连接字符串

数组

  • 可包涵任意类型的元素，通过索引访问。长度直接访问length属性。
  • 重新给length属性赋值，会导致数组长度变化，要么截取要么添加undefined
  • 可以通过索引来改变数组中对应的元素值

  • 使用上一条出现越界时，同样会改变数组，使用undefined来补位
    在使用数组时，不提倡越界，修改数组长度

  • indexOf(10) [返回数组中对应元素的索引;找不到返回－1]

  • slice(a,b) 与substring()对应，同样是左闭右开, [返回其子数组]
  • slice(a) 从索引a开始直到结束, [返回所要求的数组]
  • slice() 直接返回一个数组的拷贝
  • push/pop 向数组末尾添加或删除元素。[push返回值为数组大小]，[pop返回元素]
  • unshift/shift 向数组头部添加或删除元素 [unshift返回数组长度，shift返回元素]
  • sort() 对当前数组排序，默认升序;[对当前数组进行修改]
  • reverse 反转数组
  • splice(offset, count, ……) 表示从offset位置删除count个元素，然后从offset位置加入若干元素。第三个参数可以缺省。[返回的是被删除的元素]
  • concat() 参数可以是单独的元素，也可以是数组，无限制，拆开之后重新组装为新数组并[返回新数组]
  • join(‘-‘)表示将数组中的每一个元素用指定字符连接，然后返回字符串。若数组中有非字符串，转换后连接 ［返回字符串］

多维数组

数组中的某些元素也是数组。

对象

  • 无序键值对集合
  • 取出之后可以重新赋值
  • 如果是使用的‘’包裹的非标识符键，则需要使用[“xxx”]来获取键值
    • 其他情况均通用
  • 添加属性 给一个不存在的属性赋值
  • 删除属性 delete 对应属性
    • 删除一个不存在的属性 不报错
  • in 判断属性是否存在 name in person
    • 当属性为其继承“父类”的属性时，会返回true，如Object的toString
    • 因此给出了更加合理的方法：name.hasOwnProperty(…);只在自己的属性中查找

条件判断

基本规则和java以及其他语言一样，都建议加上{}

循环

  • for…in 可用来遍历一个对象中所有的属性。如果给数组做for…in，则返回遍历的是下标。如果使用属性赋值，则该属性会被加到数组中，而for…in会将它包含在内。数组的length属性却不会改变。
  • do while / while

Map Set

  • Map:
    • 可以通过一个二维数组作为参数来实例化,或空集合
    • set(key, value);//存入一个键值对
    • has(key);//判断是否存在，[返回true／false]
    • get(key);//获取值，返回［值］
    • delete(key);删除对应的key值对,如果删除成功，［返回结果true，否则false］
    • 如果重复压入相同的键值元素，则覆盖前者
      以上存在对不存在的对象的操作，返回undefined
  • Set:
    • 相当于Map的简化版，不存储value，只存key，即就是存储不重复元素
    • 使用一维数组作为输入实例化或者直接创建空的。
    • add(key) 向数组中添加元素
    • delete(key)

Iterable

属于iterable的集合有：Array、Map、Set

  • for…of 遍历以上Iterable类型集合

    • for(var x of a)// for…in， for…of， for…each基本一样
    • 使用 for…of 则会修复 for…in的对应的bug，它只循环属于自己的元素，直接通过赋值的元素不会被遍历。

  • iterable内置的forEach方法：[里边所有变量对外边透明]

    1. forEach for Array:
      a.forEach(function (element, index, array){
      element//当前元素
      index//当前下标
      array//数组，即就是a
      });

    2. forEach for Set:
      s.forEach(function (element, sameElement, set){
      前两个参数都是元素本身//因为Set没有索引
      });

    3. forEach for Map:
      s.forEach(function (value, key, map){

      });
      以上参数可以对应缺省,但是顺序是一一对应的，缺省顺序遵循Cplus

函数

函数就是一种代码的抽象方式

函数定义和调用

  • function functionName(arguements){
    程序如果顺序执行完毕，则返回undefined，否则按照
    代码所以写返回所要求的值
    }
    functionName(arguements)是一个函数对象，functionName可以看作指向该对象的一个变量

  • 匿名函数var abs = function(arguments){
    返回值直接赋给abs;
    //vs Java: Java 中的匿名内部类主要是用来实现接口或者继承
    //来直接获取对象的。而Js也是类似，不过Js只是相当于把函数名functionName给放到 //赋值号前边了而已，调用方法的时候，依然使用funcitionName来调用
    }

  • 函数的调用：

    • 可以给参数列表后随意加参数，不过 in vain
    • 如果缺省了唯一的参数，则返回NaN（此时，形参收到undefined，返回NaN）
    • 因此要避免收到undefined
      • typeof x! == ‘number’ 判断x的类型和number是否一致
      • throw “xxx”; 抛异常
  • arguments：函数体内部自带的辅助参数
    • 永远指向传入的参数，可以看作一个数组
    • 它的存在不依赖于是否传入参数
    • 常用来辅助判断传入参数的个数
  • rest参数：//意思是额外的参数
    • 相当于专门接收额外参数的数组
    • rest在ES6之前是空的，需要我们人为为它通过arguments来push元素
    • 在ES6之后，我们可以将...rest作为参数放到形参最后，系统会自动将多余的参数存入rest，如果参数未满，rest接收空数组。

Notice

  • return :在写代码的时候，如果不在return后加分号，则Js引擎会自动加；，很容易引起歧义。可以考虑使用代码块来解决，别忘记加;喔～ 总之不要换行

变量作用域

  • 在代码块内部定义的所有变量在代码块外部是不可见的。
  • 代码块内部定义的变量，在顺序执行的前提下，在代码块内部是可见的，同样在嵌套的函数体中适用。
  • 代码块执行完毕后，其中的变量一律被释放

RecyclerView的使用与提升

发表于 2015-11-26   |   分类于 Android

前言：

自7月份参加工作以来，对于列表的实现，一直采用了RecyclerView的实现方式。一方面呢，是因为文哥之前在掘金一直用的是RecyclerView，而且在面试的时候，给我讲过这一段知识；另一方面Recycler相对于ListView有着(android studio之于eclipse级别的优势)。所以，我毅然决然地抛弃了ListView,投奔了RecyclerView的怀抱。

正文：

RecyclerView的优势：

  • 它自带ViewHolder来实现View的复用机制，再也不用ListView那样在getView()里自己写了
  • 使用LayoutManager可以实现ListView，GridView以及流式布局的列表效果
  • 通过setItemAnimator(ItemAnimator animator)可以实现增删动画（懒的话，可以使用默认的ItemAnimator对象，效果也不错）
  • 控制item的间隔，可以使用addItemDecoration(ItemDecoration decor)，不过里边的ItemDecoration是一个抽象类，需要自己去实现…

  • 代码中的写法基本和ListView相差无几，但还是要重点说一下：

    1. 在实例化RecyclerView之后，我们需要使用setLayoutManager()给它设置布局管理器，其中的实参即就是LayoutManager，这里总共有两种LayoutManager：

      1. StaggeredGridLayoutManager，是我们之前提到的流式布局：
        它有一个构造方法StaggeredGridLayoutManager(int spanCount, int orientation)，第一个是网格的列数，第二个参数是数据呈现的方向
        （如果是竖直，那么第一个参数的意义就是列数，反之为行数。而且第二个参数在StaggeredGridLayoutManager里也有同样名称的常量，请同学们自行采纳［这里还是建议大家使用库里自带的常量，因为他们一般都是整型值，这样可以避免各个类里所判别的常量值不一样而导致的其他问题］）；

      2. GridLayoutManager，网格布局（流式布局应该是它的一个特殊情况）：
        GridLayoutManager(Context context, int spanCount）或
        GridLayoutManager(Context context, int spanCount, int orientation, boolean reverseLayout)需要说明的是，最后一个参数表示的是是否逆向布局（意思是将数据反向显示，原先从左向右，从上至下。设为true之后全部逆转）。
        小提示：在这两个LayoutManager中，默认的orientation为vertical，reverseLayout为false。对应的参数在GridLayoutManager中都有对应的方法来进行补充设置。而在StaggeredGridLayoutManager中所有的方法都针对reverseLayout做了判断，然而它并没有给出这个参数设定值的api。

      3. 线性布局, LinearLayoutManager
        LinearLayoutManager(Context context)构建一个默认布局方向为VERTICAL的RecyclerView，这种样式也是ListView默认的。
        LinearLayoutManager(Context context, int orientation, boolean reverseLayout)。发现没有，线性布局和网格布局几乎是一样的。只是少了一个spanCount参数。

    getItemViewType(int position)是用来根据position的不同来实现RecyclerView中对不同布局的要求。从这个方法中所返回的值会在onCreateViewHolder中用到。比如头部，尾部，等等的特殊itemView（这里说成ViewHolder比较好）都可以在这里进行判断。

人在稀土

发表于 2015-11-15   |   分类于 稀土

来北京也有整整4个月，123天了。也就在今天，我决定离开稀土，暂时离开北京一段日子。
现在我就来讲讲我和稀土的故事〜

初识稀土

我是大学三年级，也就是今年春季开学开始接触android的。自己在学校利用零零碎碎的时间学了两三个月，大概在五月份左右的时候，经过铁哥介绍，我加入了大名鼎鼎的代码家的干货群。自己也经常在这个群里学习到很多很多的东西，一方面是因为我在做自己的DigCSDN的时候不懂的太多，更主要的是群里的小伙伴们都很热情，而且人人都是身怀绝技。

突然的某一天，在群里经过代码家了解到稀土(文哥(后文中的文哥均指代码家)所在的团队)在招实习生，通过稀土招人看到各个岗位都在招人并且招实习生的消息之后，通过文哥后来知道android也招实习生，当时萌生了一点想法。

后来等我做完DigCSDN之后，补充了一下之前发在各大招聘网站上的简历之后，抱着试试看的心态发给了文哥。大概过了一两天过后，就收到文哥的回复，说是请我过去“喝茶”。生涯第一次投简历，而且还是给自己的偶像投，重点是还过了！真的很开心很开心。具体内容可以看我的这篇博客 面稀土，战码家。

身在稀土

我是7月15号到的北京，当时是和Andy一起到的北京,也算是相依为命了吧。他是稀土的web前端实习，听他说之前在学校干过一段时间的外包，期间还赚了一笔钱，买了电脑～总之在我当时看就是很厉害，毕竟相比之下，我只是一个毫无经验可言的毛头小子，新人一个！
我们两个穷学生（其实我更穷，哈哈）在北京从15号一致呆到17号三天才算是找到房子，他比我要先找到，而我没有办法只好拖着好重的行李箱最后直接前往朝阳区，根据后来的经验，大概有80分钟左右的路程，直记得当时手已经快要被磨出血了。

在这4个月的时间里，前三个月我和Andy是一起工作的。而我则是一个人在Android端坚守了三个月，这也是让我印象最深，最难熬的一段时间。因为之前没有任何的工作经验，基本算是一个小白菜鸟吧，很多东西都只听说过，从没用过，有些甚至听都没听过。几乎绝大多数时间都是在不断的学习项目中之前用到的一些库，有新需求了，自己也竭尽全力去学习相应的实现方法，而我们伟大的设计师昕霖也比较理解我，经常在设计上和我商量折中的方案。这一段时间因为没有人带着我学习熟悉项目，数不清多少次动了要离开稀土的想法，后来涛哥也一直劝我不要质疑自己出来实习的动机，自己一个人成长起来虽然苦，但是期间确实很能锻炼人的，期间提高自己的的水平是井喷式的，毕竟如果让别人带的话，自己也就负责很小的模块了，而且在实际开发的过程中，师傅也不可能腾出太多的时间来照顾我这边。

后来在第四个月的时候，稀土来了新的Android主程Next__,也就是我之前的师傅，从此展开了有人带的一个半月，期间我的任务基本就是给师傅打杂，主体功能基本由他完成，虽然具体怎么做，我也都知道，但是相比熟悉度以及考虑到时间成本的问题，一些功能模块以及之前无法解决的bug基本都是由他来解决了，小的模块就由我来写。一般只有我当天没有事情做的时候，才会让我做些大模块的开发。当出现一些比较好效果由我来写而从没接触过的时候，师傅就会帮我一起查好资料或者直接把查好的URL发给我，帮助我学习。

在进入11月，也就是我辞职的最后半个月的时候，我们的昕霖嫂子因为要待产了，所以我又尝试了一段没有产品&&设计的日子，开发进度几乎停滞了下来。每天除了解bug就是解bug，没有bug的时候，就只好自己做一些事情（比如刷一下《程序员代码面试指南》，写点小清新的文章之类的）。印象最深的就是有一天从中午开始就断网了，那天下午就一直没有来过网，只好自己翻开书看起了《Android开发艺术探索》，那天因为无法工作，下班的也比较早。直到后来因为学校需要考期末考试，而且一考就是一周，再加上因为前几天交了学费，自己已经承担不起当月的房租了，只好狠下心来违约房子，赔了押金不说，还给室友赔了750（T_T,好伤心）。不多说了，谁让我是一个好人呢？？？

致谢稀土的每一个人

代码家

因为文哥，我了解到了稀土，也是因为文哥，我比较轻松地拥有了这份实习offer（具体详情请见面稀土，战码家）。我知道单凭自己的能力去获取一份实习offer并不容易，至少从旅途上可以让我掉一层皮，后来才知道文哥也是在我身上赌了一把，事实证明他赌赢了。在我单独开发掘金的这三个月里，我并没有惹出什么乱子来（大家别笑话），而且根据产品设计要求，顺利跟上了iOS的脚步完成了对应1.3.5 —— 2.2.0的开发。功能上虽然没有特别大的改变，但我也基本上从整体上完成了对代码的View，所写的模块也都按照现有的框架完美兼容。
另外你能想象自己就坐在偶像面前面试的感觉吗？哈哈，那种感觉很难再有了。本来是想去上海薄荷的，因为奇哥向CTO申请实习名额未果后，只好放弃了当初去上海的想法。北京这边呢，文哥在面试的过程中就教了我很多很多的东西，很难想象如果能够和文哥一起工作三四个月，那该是多么爽歪歪的一件事情……

明明

我在稀土认识的第二个人就是明明，很多时候想要叫他明哥，但是他也一直强调这样不太好，毕竟大家都是90后，一直让我叫他明明。也是在文哥给我技术面试完毕之后，在CEO面试中认识的，整个人很坦诚，而且对于一个产品有着独到的见解，总之就是那种让人无法挑刺的观点和看法。事实证明我的看法没有错，因为在后来的同学面试阿里的消息中得知，我了解到了在阿里的产品面试中就有着QQ vs 微信 二者的差异，而标准答案和明明给我讲的基本一致，结果我想说的只有两个字“牛逼”！！！在后期的开发中，明明也参与了不少和产品工作上的事情。作为剑桥的高材生，对于一些事情的看法和观点真的是不一般，这点给256个赞！！！

明泽

他的id是ming_Zhe,关于为什么不是ming_ze，他说是因为从美学角度上来讲ooxx……哈哈，大家不要误会，我是想不起来了。明泽作为创始人之一，虽然担任的是掘金的ios端的开发以及财务行政，但毫无疑问成为了我前三个月的师傅。哈哈，一方面呢，是从一些移动端开发的经验上来讲，另一方面就是比较有意思的一件事情。在来稀土之前，我只会用Github做一些add, commit, pull, push这几个基本的操作，而现在我可以用到的命令多达20余个，而且不只是简单的记忆，更多的是项目中实际的应用。现在只有一些比较生僻的命令还没有用到，基本上都可以熟练使用了。这都得归功于好几天下班以后以及日常明泽和明明两个人的“Git教程时间”，而且在这次离职的过程中，太多的事情都做的过于草率，以至于我都回到学校了，他才知道。他也给了我一些批评。显而易见，明泽教给我的不只是一些知识上的东西，更多的是作为职场分子的一些经验！感谢有这样的老师……

江江

群里的一些朋友都叫他江大大，因为牛逼的市场公关运营能力以及前端开发能力(突然发现稀土的四位创始人都是前端大牛，因此程序员全栈化不得不再次摆上我个人的日程表)。尤其是最近在处理“蜻蜓FM造假门事件”上处理的游刃有余，让人不得不折服。他这个人执行力特别强，记得那次吃饭的时候，在谈到我经常用于查看的一个手册工具“Android学习手册”的时候，他说到要用一些黑科技做一些事情，我就感觉到不对劲了，然后商量着我和Next给做一个类似于该功能的app，而且一路还讨论了不少。因为考虑到可能会和我们当前的开发任务会有冲突，他就说他用phoneGap做一个,结果呢？就在第二天，我就在他电脑上看到了一个初具模型的web app，这执行力简直也是没谁了！！！

在稀土的每一天，都在不断的进步中。这次情非得已的离开的确让我失去了这份难得的机会，失去了在稀土的每一个人。但是还是那句话：”天下哪儿有不散的宴席“，或许这次的离开是为了下次更好的重逢做准备也说不定呢～～～
